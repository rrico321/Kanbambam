**1. Create apps/cli/src/commands/board/create.ts (BORD-01, BORD-05, D-01, D-02, D-04):**
Export `boardCreateCommand(name: string, options: { workspace?: string }, globalOptions: GlobalOptions): Promise<void>`

- Resolve workspace ID: `const workspaceId = resolveWorkspaceId(options)` — throws if no workspace context/flag (per D-04)
- Create board: `apiRequest(`/api/v1/workspaces/${workspaceId}/boards`, { method: 'POST', body: JSON.stringify({ name }) })`
- If board creation fails: show error and exit
- Parse board response as `ApiEnvelope<BoardResponse>`
- Create 3 default columns sequentially (BORD-05 — order matters for position assignment):
  ```typescript
  const defaultColumns = ['To Do', 'In Progress', 'Done']
  const failedColumns: string[] = []
  for (const colName of defaultColumns) {
    try {
      await apiRequest(`/api/v1/boards/${board.id}/columns`, {
        method: 'POST',
        body: JSON.stringify({ name: colName }),
      })
    } catch {
      failedColumns.push(colName)
    }
  }
  ```
- Also check response.ok on each column create (not just catch). If column create response is not ok, add to failedColumns.
- Success output per UI-SPEC:
  - If no failures: `Created board {name} ({id}) with columns: To Do, In Progress, Done`
  - If some failures: `Created board {name} ({id}). Warning: failed to create default column(s): {failedColumns.join(', ')}`
  - JSON: `outputJson(board, { ...meta, defaultColumns: failedColumns.length === 0 ? ['To Do', 'In Progress', 'Done'] : { created: defaultColumns.filter(c => !failedColumns.includes(c)), failed: failedColumns } })`

**1. Create apps/cli/src/commands/board/rename.ts (BORD-03, D-02):**
Export `boardRenameCommand(id: string, options: { name?: string }, globalOptions: GlobalOptions): Promise<void>`

- Validate `options.name` provided, error if not: `"Name is required. Use --name <new-name>"`
- Call `apiRequest(`/api/v1/boards/${id}`, { method: 'PATCH', body: JSON.stringify({ name: options.name }) })`
- Handle 404 per UI-SPEC: `Board not found. It may have been deleted. Run kanbambam context clear if your context is stale.`
- Success per UI-SPEC: `Renamed board to {data.name} ({data.id})`
- 3-mode output (json/ink/plain)

**2. Create apps/cli/src/commands/board/delete.ts (BORD-04, D-03):**
Export `boardDeleteCommand(id: string, options: { force?: boolean }, globalOptions: GlobalOptions): Promise<void>`

- Always GET board first: `apiRequest(`/api/v1/boards/${id}`)` to validate existence and get name
- If not `--force`: run confirmation per UI-SPEC: `Delete board "${name}" (${id})? This will delete all columns and items. [y/N]`
- If confirmed or `--force`: `apiRequest(`/api/v1/boards/${id}`, { method: 'DELETE' })`
- DELETE returns 204. Success per UI-SPEC: `Deleted board {name} ({id})`
- JSON: `outputJson({ deleted: true, id }, {})`
