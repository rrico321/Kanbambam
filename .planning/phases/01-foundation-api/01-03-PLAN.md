**Workspace routes** -- 5 endpoints per D-06 nested REST and D-08 versioning:

All routes under `/api/v1/workspaces`. User ID extracted from `c.get('jwtPayload').sub`. All queries scoped to the authenticated user's workspaces only.

1. `GET /api/v1/workspaces` -- List workspaces (paginated per D-10)
   - Query params: `cursor` (optional string), `limit` (optional int, default 20, max 100)
   - Query: `SELECT * FROM workspaces WHERE user_id = $userId AND (id > decodeCursor($cursor) IF cursor) ORDER BY id ASC LIMIT $limit + 1`
   - Response: `{ data: Workspace[], meta: { total: number, cursor: string | null } }` per D-04
   - Fetch `limit + 1` rows. If `results.length > limit`, hasMore=true, nextCursor = encodeCursor(lastItem.id)
   - Total count: separate `SELECT COUNT(*)` query

2. `POST /api/v1/workspaces` -- Create workspace
   - Body: `{ name: string }` validated by `CreateWorkspaceSchema`
   - Generate ID: `generateId('workspace')`
   - Insert into `workspaces` table with userId from JWT
   - Response: 201 `{ data: Workspace, meta: { requestId } }` per D-04

3. `GET /api/v1/workspaces/:id` -- Get single workspace
   - Param: `id` (string, validated as starting with `wks_`)
   - Query with ownership check: `WHERE id = $id AND user_id = $userId`
   - If not found: 404 `{ error: { code: "WORKSPACE_NOT_FOUND", message: "Workspace not found" } }` per D-05
   - Response: `{ data: Workspace, meta: { requestId } }`

4. `PATCH /api/v1/workspaces/:id` -- Update workspace
   - Body: `{ name: string }` validated by `UpdateWorkspaceSchema`
   - Update with ownership check, set `updatedAt` to `new Date()`
   - If not found: 404
   - Response: `{ data: Workspace, meta: { requestId } }`

5. `DELETE /api/v1/workspaces/:id` -- Delete workspace (hard delete per D-09)
   - Delete with ownership check
   - If not found: 404
   - Response: 204 No Content

**Board routes** -- 5 endpoints under `/api/v1/workspaces/:workspaceId/boards` per D-06:

5. `DELETE /api/v1/boards/:id` -- Delete board (cascades to columns and items per D-09)

**Column routes** -- 5 endpoints:

Columns belong to boards. Ownership chain: column -> board -> workspace -> user. Must verify the chain back to the authenticated user.

1. `GET /api/v1/boards/:boardId/columns` -- List columns for a board
   - Verify board ownership (board -> workspace -> user)
   - Query columns WHERE board_id = $boardId, ORDER BY position ASC
   - Cursor pagination per D-10 (same pattern as workspaces)
   - Response: `{ data: Column[], meta: { total, cursor } }`

2. `POST /api/v1/boards/:boardId/columns` -- Create column
   - Body: `{ name }` validated by `CreateColumnSchema`
   - Verify board ownership
   - Generate ID: `generateId('column')`
   - Generate position: query the last column's position in this board (`SELECT position FROM columns WHERE board_id = $boardId ORDER BY position DESC LIMIT 1`), then call `appendPosition(lastPosition)`. If no columns exist, `appendPosition(null)`.
   - Insert column
   - Response: 201 `{ data: Column, meta: { requestId } }`

3. `GET /api/v1/columns/:id` -- Get single column
   - Verify ownership chain (column -> board -> workspace -> user)
   - 404 if not found
   - Response: `{ data: Column, meta: { requestId } }`

4. `PATCH /api/v1/columns/:id` -- Update column (name and/or position)
   - Body: `UpdateColumnSchema` (name optional, position optional)
   - Verify ownership chain
   - If position is provided, update it directly (client computes the fractional index or sends a desired target). Alternatively, accept `afterId` and `beforeId` for server-computed position -- use `insertBetween(beforePosition, afterPosition)`.
   - Set `updatedAt`
   - Response: `{ data: Column, meta: { requestId } }`

5. `DELETE /api/v1/columns/:id` -- Delete column (hard delete per D-09, cascades items)
   - Verify ownership chain
   - 404 if not found
   - Response: 204

**Item routes** -- 6 endpoints:

Items belong to columns. Ownership chain: item -> column -> board -> workspace -> user.

1. `GET /api/v1/columns/:columnId/items` -- List items in a column
   - Verify column ownership chain
   - Query items WHERE column_id = $columnId, ORDER BY position ASC
   - Cursor pagination per D-10
   - Response: `{ data: Item[], meta: { total, cursor } }`

2. `POST /api/v1/columns/:columnId/items` -- Create item
   - Body: `{ title, description? }` validated by `CreateItemSchema`
   - Verify column ownership chain
   - Generate ID: `generateId('item')`
   - Generate position: `appendPosition(lastItemPosition)` -- query last item position in this column
   - Insert item with columnId from path
   - Response: 201 `{ data: Item, meta: { requestId } }`

3. `GET /api/v1/items/:id` -- Get single item
   - Verify ownership chain (item -> column -> board -> workspace -> user)
   - 404 if not found
   - Response: `{ data: Item, meta: { requestId } }`

4. `PATCH /api/v1/items/:id` -- Update item (title, description, dueDate, labels)
   - Body: `UpdateItemSchema` (all fields optional)
   - Verify ownership chain
   - Set `updatedAt`
   - Response: `{ data: Item, meta: { requestId } }`

5. `DELETE /api/v1/items/:id` -- Delete item (hard delete per D-09)

   - 404 if not found

   - Response: `{ data: Item, meta: { requestId } }`
