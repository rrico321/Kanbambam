**1. Create apps/cli/src/commands/workspace/rename.ts (WKSP-03, D-02):**
Export `workspaceRenameCommand(id: string, options: { name?: string }, globalOptions: GlobalOptions): Promise<void>`

- Validate `options.name` is provided. If not, error: `"Name is required. Use --name <new-name>"`
- Call `apiRequest(`/api/v1/workspaces/${id}`, { method: 'PATCH', body: JSON.stringify({ name: options.name }) })`
- Handle error responses (404 = entity not found per UI-SPEC error copy: `Workspace not found. It may have been deleted. Run kanbambam context clear if your context is stale.`)
- Success output per D-10 and UI-SPEC:
  - JSON: `outputJson(data, meta)`
  - Ink: render `SuccessMessage` with `Renamed workspace to {data.name} ({data.id})`
  - Plain: same text via outputPlain

**2. Create apps/cli/src/commands/workspace/delete.ts (WKSP-04, D-03):**
Export `workspaceDeleteCommand(id: string, options: { force?: boolean }, globalOptions: GlobalOptions): Promise<void>`

- If NOT `options.force`: First fetch workspace name via `apiRequest(`/api/v1/workspaces/${id}`)` (GET). Then run confirmation per UI-SPEC: `confirm(`Delete workspace "${name}" (${id})? This will delete all boards, columns, and items.`)`. If declined, exit without action.
- If confirmed or `--force`: call `apiRequest(`/api/v1/workspaces/${id}`, { method: 'DELETE' })`
- DELETE returns 204 (no body). Success output per D-10 and UI-SPEC:
  - JSON: `outputJson({ deleted: true, id }, {})`
  - Ink: render `SuccessMessage` with `Deleted workspace {name} ({id})`
  - Plain: same text via outputPlain
- For --force mode where we skip the GET: we don't have the name. Use the id in the message: `Deleted workspace ({id})`
  - Better approach: always GET the workspace first (validates existence and gets name), THEN confirm, THEN delete. This way --force skips only the prompt, not the name fetch.
