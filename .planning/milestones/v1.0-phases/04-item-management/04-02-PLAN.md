**edit.ts** — Export `itemEditCommand(id: string, options: { title?: string; description?: string; label?: string[]; dueDate?: string }, globalOptions: GlobalOptions): Promise<void>`

Per D-04, D-05, D-07, D-09:

1. Build PATCH body from provided flags only — omit unspecified fields:
```typescript
const body: Record<string, unknown> = {}
if (options.title !== undefined) body.title = options.title
if (options.description !== undefined) {
  body.description = options.description === '' ? null : options.description  // D-07: empty string clears
}
if (options.label !== undefined && options.label.length > 0) {
  // D-04 replace-all semantics
  if (options.label.length === 1 && options.label[0] === '') {
    body.labels = null  // --label '' clears all
  } else {
    body.labels = options.label  // replace-all with new labels
  }
}
if (options.dueDate !== undefined) {
  body.dueDate = options.dueDate === '' ? null : options.dueDate  // D-05: empty string clears
}
```

2. Validate at least one flag provided — if `Object.keys(body).length === 0`, output error: "No changes specified. Use --title, --description, --label, or --due-date."

3. Validate due date format if provided and non-empty: check `/^\d{4}-\d{2}-\d{2}$/` regex. If invalid, error: "Invalid date format. Use YYYY-MM-DD (e.g., 2026-04-01)."

4. `PATCH /api/v1/items/${id}` with `JSON.stringify(body)`

5. On success: output `Updated item "${data.title}" (${data.id})` in plain/ink, full data in json mode

6. On error (non-ok response): parse ApiError, display via ErrorDisplay pattern. 404 = "Item not found."

**Commander label collection function** (defined at module top):
```typescript
function collect(value: string, previous: string[]) {
  return previous.concat([value])
}
```
The Commander option definition will be: `.option('-l, --label <tag>', 'Label (repeatable, replaces all)', collect, [])`

Per D-11 (delete confirmation), follow the GET-first pattern from workspace/board/column delete:

1. `GET /api/v1/items/${id}` — fetch item to get title for confirmation display
2. If `!response.ok`: error "Item not found." for 404, else parse ApiError
3. If `!options.force`: call `confirm(\`Delete item "${data.title}"? [y/N]\`)` — if not confirmed, return silently
4. `DELETE /api/v1/items/${id}`
5. On success: output `Deleted item "${data.title}" (${id})` in plain/ink, `{ deleted: true, id }` in json mode
6. On error: parse ApiError, display via ErrorDisplay pattern
