# Phase 5 — UI Design Contract

> Visual and interaction contract for the Trello-style web app. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Tailwind components, consistent with existing auth pages) |
| Icon library | Lucide React (`lucide-react`) -- lightweight, tree-shakeable, consistent with modern React ecosystem |
| Font | Inter (Google Fonts) -- already configured in `layout.tsx` with weights 400, 600 |

**Note on shadcn:** The project has 5 existing hand-rolled Tailwind components from Phase 2 (AuthCard, FormField, SubmitButton, ErrorAlert, AuthLink). Introducing shadcn mid-project would create visual inconsistency. Continue with the established Tailwind-only pattern.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label chip inline padding, badge spacing |
| sm | 8px | Card internal padding between elements, compact list gaps |
| md | 16px | Card padding, column gap between cards, sidebar item padding |
| lg | 24px | Column header padding, modal section spacing |
| xl | 32px | Board horizontal padding, sidebar width padding |
| 2xl | 48px | Modal max-width gutter |
| 3xl | 64px | Sidebar collapsed-to-expanded transition width |

Exceptions:
- Sidebar expanded width: 256px (16 * 16, not on scale but standard sidebar width)
- Sidebar collapsed width: 48px (icon-only rail)
- Column width: 280px (fixed, Trello standard)
- Card drag handle touch target: 44px minimum (accessibility)

## Component Visual Contracts

```
Container: bg-white dark:bg-gray-800 shadow-sm rounded-lg border border-gray-200 dark:border-gray-700 p-3
```

### Column (D-01, D-03)
```

```

## Interaction Contracts

### Drag and Drop (D-07, D-08)

| State | Visual |
|-------|--------|
| Idle | Card at full opacity, default shadow |
| Dragging (source) | Card at 50% opacity (ghost), cursor: grabbing |
| Drop target (between cards) | 2px blue line (`border-blue-400`) at insertion point |
| Drop target (empty column) | Column border becomes dashed blue |
| Drop complete | Instant position update (optimistic), API fires in background |
| Drop error | Card snaps back to original position, error toast via sonner |

### Click-to-Edit (D-10)

| State | Visual |
|-------|--------|
| Display | Text with `cursor-pointer`, subtle hover background (`hover:bg-gray-100 dark:hover:bg-gray-700 rounded px-2 py-1`) |
| Editing | Input/textarea replaces text, auto-focused, matching font size and weight |
| Save | Blur or Enter commits, fires API PATCH, optimistic update |
| Cancel | Escape reverts to original value, exits edit mode |
| Empty | Save rejected, reverts to previous value (title cannot be empty) |

### Modal Lifecycle (D-09)

| Trigger | Behavior |
|---------|----------|
| Open | Click card anywhere (except drag handle if separate) |
| Close | Click overlay, click X button, press Escape |
| Animation | None for v1 (keep simple) |
| URL | Modal open does NOT change URL (avoids routing complexity for v1) |

## Loading States

| Context | Pattern |
|---------|---------|
| Board loading | Skeleton: 3 columns x 3 cards each. Column = `w-[280px] h-10 rounded-lg bg-gray-200 dark:bg-gray-700 animate-pulse`. Card = `w-full h-20 rounded-lg bg-gray-200 dark:bg-gray-700 animate-pulse`. |

## Responsive Behavior

| Breakpoint | Layout |
|------------|--------|
| >= 1024px (desktop) | Sidebar visible (collapsible) + board view with horizontal scroll |
| 768-1023px (tablet) | Sidebar collapsed by default, toggle to expand as overlay |
| < 768px (mobile) | Sidebar hidden (hamburger menu), columns stack vertically or single-column scroll |

**v1 priority:** Desktop-first. Tablet functional. Mobile basic usability (not polished).

## Dark Mode Implementation

| Property | Value |
|----------|-------|
| Library | next-themes 0.4.6 |
| Strategy | `data-theme` attribute on `<html>`, detected by Tailwind `@variant dark` |
| Default | System preference |
| Override | Manual toggle (Sun/Moon icon), persisted in localStorage by next-themes |
| CSS setup | `@variant dark (&:where([data-theme="dark"] *));` in `globals.css` |
| SSR flash prevention | `suppressHydrationWarning` on `<html>`, next-themes blocking script |
